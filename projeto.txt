Requisitos do sistema:

php com laravel e dbeaver utilizando as melhores práticas. O sistema é para gerenciar os veículos de uma frota, e tem as seguintes telas:
1. login: o usuário informa login e senha para ir para a tela inicial. Essa tela de login já deve reconhecer se o usuário é admin ou P4. 
2. na tela inicial: se o usuário logado for "admin" ele pode abrir a tela da lista de usuários(OPM) e fazer pesquisa, criar, editar e excluir usuário de todas as OPM. Se o usuário for P4 ele pode realizar essas interações apenas na unidade dele, e nesse caso a tela inicial deixará habilitada para alterações apenas a unidade respectiva ao P4 logado.
3.lista de viaturas: se o usuário logado for "p4" o sistema lista apenas as viaturas da OPM deste usuário e ele apenas poderá pesquisar e editar estas. Se o usuário logado for "admin" ele poderá listar todas as viaturas de todas as unidades e editar todas elas.
4. Cadastro de viatura: se o usuário logado for "p4" ele pode apenas editar os campos prefixo, trabalho e observação da entidade viatura. Se o usuário logado for "admin" ele poderá editar todos os campos da entidade viatura.
Os campos da entidade viatura são: marca/ modelo, ano fabricação, situação carga, OPM, cidade/município, emprego, tipo uso, layout, tipo veículo, tração, combustível, ativo/processo descarga, placa, área, prefixo, chassi, renavam, categoria, aquisição de dados, entrega de dados OPM, número série rádio.
Será utilizado para este projeto o servidor nginx. Dentro do projeto já deixe as configurações prontas para ele.
